Lesson 1: The Web and HTML

What is the web?

The world wide web is the world's largest computer network and collection of HTML documents. It is the result of many computers communicating with one another.

When you open a webpage, your computer sends a HTTP Request to a Server that hosts the HMTL Documents for that site. The Server then sends the HTML Documents back to your computer where a Web Browser interprets and displays the document on your screen.

This video provides a quick explanation.

HTML

HTML stands for Hyper Text Markup Language. HTML documents are the basis for almost every webpage. Each document contains text content describing "what you see" and markup describing "what it looks like and how the webpage is arranged". These documents can also refer to other documents like images, video, and music. Hyperlinks or simply 'links' to other pages can be included as well; making the web, weblike.

HTML documents are the structure of a webpage and communicate to the browser how the elements are to be arranged and displayed.

This video sums it up nicely.

Tags and Elements

HTML documents are made of multiple HTML elements. These elements are individual components that can have their attributes specified to display what the programmer wants to convey. Programmers use HTML tags, the keywords that define how a browser must format and display content, to convey these messages.

Tags are defined by an opening tag, the content you want to show, and a closing tag. Each line of HTML can contain multiple elements to present what the programmer wants to display.

This image is a great example of an 'h1' tag and how weblike the web can be by clicking on the link, as discussed in the previous topic.

Some tags don't require a closing tag. They are called void tag. Google's Styleguide states that you should not close void tags. There is no set rule or thought regarding the closing a void tag

This video breaks it down.

Computers are Stupid, Syntax is Law

Computers do not have the human ability of abstract thinking, therefore they interpret instructions a programmer communicates literally. A small mistake or typo by the programmer can lead to huge problems in a program. You must write with the correct syntax for the computer to produce exactly what you want to display.

Syntax is the spelling and grammar of a programming language. The computer expects you to type in the exact form of the language you are writing. That exact form is called the syntax of the programming language.

A good practice is to close appropriate tags as soon as possible.

Inline Vs. Block Elements

HTML elements are either inline or block.

Block elements create an "invisible box" around the content allowing you to change height and width of content, as well as many other attributes. Inline elements are not as fancy. They typically only contain text and other inline elements.

The syntax regarding these elements is vast and there are no strict rules due to the context they may be used, however you can think of block elements as structural elements and inline elements as text-based elements.

Whitespace

In HTML all whitespace will turn into a single space. Tabs, new lines, or multiple lines will not be affected unless represented by the proper tag is used to tell the browser what to display.

Lesson 2: Creating a Structured Document with HTML

Think of a Webpage as a House

HTML documents define the structure of a website. On a blueprint, the individual components of a home structure are defined by names; living room, roof, balcony, etc. The same idea applies to HTML documents. When the individual components of a webpage structure are defined, using 'span' or 'div' tags for example, you can apply style to the specific individual components you have named.

CSS stands for Cascading Style Sheets. It is a language used for defining the style of a webpage. As blueprint would define the dimensions, angles and appearance of the home structure, CSS defines the color, layout, font, and many other attributes of a webpage. It allows adaptation to different devices, screen sizes, or printers. CSS is independent of HTML and can be used with any XML-based markup language.

JavaScript Files are the interactive components of a webpage. In this example, the blueprint has been folded up, our structure and style are complete. Now it's time to add the interactive components, like furniture and appliances to our home. Some examples of interactive components on a webpage are popup windows, jump menus, alert boxes, and image rollovers. Javascript adds automation, animation, and interactivity to webpages.

These are the three programming languages that go into web development.

Document Object Model

The Document Object Model or DOM is a way to represent and interact with a webpage. The nodes or HTML elements of the document are organized into a tree structure, called the DOM tree. The nodes in the DOM tree are organized starting with the '!DOCTYPE html' node and continuing down as the page is structured.

Boxes, Boxes, Boxes

Everything you see or interact with on a webpage is composed of a series of boxes. All the elements that create a webpage are in rectangular boxes. Thinking of items on a webpage in terms of boxes makes programming easier. There is much to learn about this concept and we will cover it in Lesson Three.

Right now, just be aware of this concept and know that it is important when understanding web development.

The Structure of an HTML Document

When reading an HTML document in Google's Developer Tools, for example, you will see indentations on the left side of the document before an element begins. The further indented an element is, the deeper it is nested in the corresponding box.

Lesson 3: CSS Style and HTML Structure

DRY, "Don't Repeat Yourself...or else!"

A key principle of software development is DRY. It's an acronym meaning "Don't Repeat Yourself.". The purpose is to reduce repetition of information, making software systems easier to control.

In CSS this means using reusable classes for elements that are going to have the same style, rather than writing the style for each element. The benefit of this concept is clear, however there is much written about this if you want an in depth explanation.

Inheritance

Inheritance is a mechanism which applies properties to an ancestor element and it's descendant elements as well. Descendant elements are the elements enclosed by ancestor elements. There are no specific descendant or ancestor element, their status is simply determined by their placement in the DOM. Descendant elements can inherit the CSS properties from any ancestor element enclosing them. Generally, descendant elements inherit text-related properties such as font, line breaks, and color among others. Box-related properties such as height, border style, padding etc. are not inherited.

Inheritance prevents certain properties from being repeated, allowing programmers to write less CSS. There are opposing thoughts to allowing inheritance in CSS. Just like closing void tags, there is no set rule or practice regarding this as well.

Selectors

In CSS you first write a selector. When you write HTML, you give class names to the parts of the document you will apply style to or modify. These class names are the selector.

There are various ways to identify selectors. This article provides an great explanation of selectors.

The Box Model

Each HTML element is a box that contains four components. These components will be described as Google Chrome represents them within developer tools. They are as follows:

Content, the blue box - This is the image or text appearing on a webpage. It is the nucleus of an HTML element and sits at the center of the box. The other components protect it so that other elements don't overlap or sit too close to the element .

Padding, the green box - This clears an area around the content and is affected by the background color of the box. It's the space inside the box that protects the content directly.

Border, the yellow box - Surrounds padding and content. Border is inherited from the color property of the box.

Margin, the orange box - This clears an area around the border, it's transparent. Basically the space between boxes.

Box Size

When you are sizing boxes there are two techniques for dealing with sizing issues.

- Set box sizes in terms of percentage rather than pixels. This will display the content depending on the size of the screen. When set by pixels, the content will remain that exact size, regardless of screen size.

- Set box-sizing attribute to border-box. This makes the sum of the element size include the border and padding, making it easier to make layouts for the webpage.

By setting a max width for a box, the box can become smaller if necessary, but never go larger than specified.

Box Positioning

Div tags are block elements. By default they will take the entire width of a page. Adding the rule "display: flex;" to the appropriate selector will override the default and allows div tags to appear next to each other.

Code, Test, Refine

Continue to test and refine your code until it looks like the design mock-up.

- Get the structure of the page right. This can be done by visually "boxifying". Then move on to tags. If something needs to be fixed, fix it, like headers for example.

- Think about the size of the boxes. How big should they be?

- How do you want the boxes to be positioned on the screen? Make the changes necessary to do this.

- Once the layout is complete, look at the smaller details. These can be polished until the webpage looks like the design given.

It is important that HTML and CSS are correct and comply with standards. If incorrect, the display may be correct in one browser and wrong in another. There are HTML and CSS validators available to test your code.
